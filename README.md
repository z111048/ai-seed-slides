## 核心痛點

都市設計審議報告書審查涵蓋章節完整性、書圖一致性、法規檢討等項目，規則明確但耗時。科室 7 名承辦人每年消耗約 **200 人日**於重複性格式比對，每次補正複查耗時約半天（4 小時）。

---

## AI Agent 解決方案

以 AI Agent 自主執行審查工作流程，Agent 接收報告書後調用工具鏈輸出結構化補正意見，承辦人專注實質審查決策。

**Agent 工具鏈（4 個介入節點）：**

| 節點 | Agent 行為 | 工具 |
|------|-----------|------|
| 掛件 × 送件 | 格式自動檢核，減少退件 | OCR 解析 + LLM 格式檢查 |
| 書圖審查 | 補正意見草稿生成 | RAG 歷史意見庫 + LLM 撰稿 |
| 審查會議 | 會議逐字稿 + 決議摘要 | 語音辨識 + LLM 摘要 |
| 核定作業 | 版本差異追蹤，確認修正回應 | Diff Engine + LLM 驗證 |

採**先訂閱 → 再客製**策略，POC 以 Claude.ai / Copilot 訂閱制起步，不綁 API 帳單。

## 操作方式（桌面版）

- **← → / Space**：切換投影片
- **右側圓點**：直接跳頁（共 10 個圓點）

- **← 鍵**：重新播放
